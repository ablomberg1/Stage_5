Stage 1: The Basics of the Web and HTML

Lesson 1: Webpages, Documents, and Structures

Basics of the Web and HTML

HTML hyper text markup language is a computer language used to create web pages on the Internet. The Internet uses HTTP to transmit data to and from servers. HTML is the heart of the web and includes documents is made up of text content (what you see), markup (what content looks like), references to images and videos, other documents and links to other pages.

HTML is made up of elements. An element consists of <opening tags> content and <closing tags>. Remember, computers are not stupid they are sensitive. Computers do exactly as the computer programmers tell them.

HTML Attributes

HTML Attributes are within a tag and a tag may have multiple attributes <tag attr="value">contents</tag>. The attribute <a> is an anchor tag. Anchors are for making links. href is an attribute that indictates the URL being linked. Example <a href=”www.reddit.com”> derp</a> creates derp on the web page. Selecting the link sends the user to the url www.reddit.com.

Another attribute is <img> which is used for including images, example: <img svc=”url” alt=”text”>. The alt=”text” piece is required for broken requests. The alt attribute is also there for those who are blind. Example: . The img attribute does not require a closing tag because it has no content.

Whitespace

There are Inline and Block elements. br = break in the line is an Inline element, p = paragraph which is used around the text and is a Block element. Text can be on one or multiple lines and is dependent on which type of tag is used.

Inline: <br>, <em> (italic), <img>, <a>, <strong>
Block: <span>, <p>, <div>, <form>
Container Elements: store stuff inside them

<Span> represents an inline element. Span is like words within a sentence. All text remains in one line. But when changed to div the content is now block.

Structure of an HTML Document

An HTML Document has a basic structure with a head which includes the meta-data, java script, css and title. The body of the document contains all the data and images, what is considered the insteresting stuff, the contents of the document. These two sections are found within the opening and closing html tags.

<!DOCTYPE HTML> designates what kind of HTML is used and is listed before the opening html.

Lesson 2: Create a Structured Document with HTML

HTML, CSS and Java

HTML is the structure; syntax, rules of the page. CSS is the style, how it looks. Java is the interactive components. HTML controls the structure of a page using DOM, a tree like structure. HTML files are like the structure of a house. When looking at the structure you are determining where the walls go, what size are the rooms, where does the kitchen go? CSS is the style of the document, how things look. In the house example this would be the pictures on the wall, colors of the floor and walls. Java script files are interactive components like a garage door opener and tv remote. These change some element of your house.

There are rules for how a website behaves and many web tools to help you build websites. For example Wikipedia's inner structure can be viewed using Google Chrome. DevTools shows the structure of the page as the browser interprets it. Every element in a structure is a rectangle. If you go deep enough into a structure you will find the same data that is on the web page. the sideways triangle is an html element that could be text or another element. Web pages include images and text.

The style panel is on the right. When a line on the left side is selected the CSS style is presented on the right. The web browser knows how to display the the HTML file, data, and images by following the syntax and rules of the DOM

DOM

“DOM” (document object model) is the tree-like structure of a web page, a branching structure. DOM is a standard convention for representing or interacting with objects in HTML. HTML is made up of tags in elements which make up the tree. These elements are nested creating the branches. This is how HTML classifies page contents, tells the browser which part is text or image.

To tie structure and style together the HTML document can provide the browser with info on what style to use and where to get it. There are specific rules for how these elements work. To define style use the CSS Cascading Style Sheet language. CSS allows you to use a specific syntax and rules to change how the text looks on the web, the borders, position, font etc.

Boxes, Boxes Everywhere

In the shapes demo, right click on a web page>inspect element to view the HTML and CSS. Everything is in rectangles. If you uncheck the border radius circles become a square.

“Boxify” your design. Start with big boxes. Don’t worry about the text or images initially, work from big to small. From the boxes create HTML. Example used was My Favorite Apps, with an image and text. Class attributes are labels on boxes that remind you of what is inside. These can have several labels. Use descriptive labels to help remind you of what the box is for. For instance the top box could be “title”.

Lesson 3: Adding CSS Style to HTML Structure

Style

The structure created by My App (example in class) does not include style. Add <!DOCTYPE html>, then add <html> to indicate we are using html. Add <head> to add style <link rel=”stylesheet” type=”text/css” href=”style.css”> then close tag for the </head>. The <body> tags include the code right before the end of lesson 2 for My App. Now add style to the style.css file. “.description {color:red;}”. Save the file and refresh browser. You will see that the text in the description box turned red.

CSS stands for Cascading Style Sheets. Cascading means the most specific rule is applied to control the style of HTML documents. The rule that best describes the element is used.

Avoid Repetition

Avoiding Repetition is important to maintain consistency throughout the HTML file. The style can more easily be maintained and fewer errors are made. Repetition can also make editing difficult.

Inheritance

Inheritance is where the key feature in CSS properties are applied to an element and all its descendants. Text related properties such as color, font, letter/line spacing (and others) can be inherited, but not box related properties such as background, border, display, float & clear. Inheritance prevents repetition, and allows developers to write less CSS. Also enhances faster loading of web pages and clients save money on bandwidth.

Instead of memorizing all attributes and tags utilize the CSS Reference Tool which lists styles of HTML elements. To reduce repetition a rule can be setup to apply to every h1 element for example, or to a class. CSS can also be included in the head of your HTML, this is good for small projects.

Separate CSS File

For larger projects link the HTML file to a separate CSS file. HTML is the main.html file, CSS is the main.css file. Add a <head> tag to the top of the HTML then add a <link> tag inside the <head> element. Reminder: Link is a void tag: there is no closing tag needed. Add the following attributes to <link>

ref = "stylesheet"
type= "text/css"
href= "main.css"

The Box Model

HTML elements are boxes and each box has four components: margin, border, padding and content. Summing up the last three; border, padding and content make up the element width. Margin is not included. 'box-size:border-box' css code creates the calculation of the border size or set size of the border.

It can be difficult to get the box sizes just right. There are two techniques to help with sizing issues:

1. Use a percentage instead of pixels for size.
2. Set box sizing attribute to border-box for every element

Add browser specific prefixes before style definition. The size of the boxes are set in pixels. These can also be done in percentages.

Box Positioning

Flex-box layout is an efficient way to layout space in a container or box. Because divs are block elements they take up the entire width of the page by default. By adding the flex rule display CSS will override this behavior and let the divs appear next to each other. In the My App example the image is up against the left border and text directly to the right due to flex-box.

Code, Test, Refine

To begin get a structure of the design and visually 'boxify' the page. Create tags and headings, size the boxes, and position everything on the screen. Once a general layout is complete then begin working on the smaller details.

Divide and conquer, look for natural boxes. Look for repeated styles and semantic elements then begin writing your HTML. Apply styles, again big to small, then go back and fix any problems. Developer tools are extremely helpful, allowing the developer to change things on the fly. Make sure the mockup looks the same as the finished product. Look at the margin size and get as close to the mockup as possible.

Validation of code can be done using the Markup Validation Service and CSS Validation Service. These services will check the code for errors and provide warnings.

Lesson 1: Introduction to "Serious" Programming

Telling Computers What to Do

Computer science includes solving problems by breaking them into smaller pieces then describing the sequence of steps to solve each piece and those steps can be executed by a computer to get the best result for the given query.

There are three pieces to a search engine: find the data, build an index and rank pages. Find data is done by crawling web pages. The index is built to respond quickly to search queries and rank pages to get the best results for a given query.

Getting Started: Programming

Programs
Computer programs follow instructions. Examples would include web browsers, Python interpreter, an application on your phone, code written to complete a specific task like print "hello". Web Crawler searches the World Wide Web and produces a list of web pages, starting with a seed page, a link which is really a special kind of text on that web page. Within a web page are more links to more pages, and this continues building a collection of data.

Programming
Python is a high level language. The Python program is an interpreter that runs programs and interprets them. Programming is the core of computer sciences. Most computers are built to do one thing but without programming a computer cannot do anything. Computers CAN be programmed to do anything you can imagine by following a precise sequence of steps. The program provides a precise set of steps which tells the computer exactly what to do.

Interpreter vs Compiler
A compiler does all the work first then runs the program. An interpreter does everything at once

Getting Started with Python

Python can be run directly using a web browser. Type the code, click run and a Python interpreter converts your code into a set of instructions for the computer to execute. These results are shown at the bottom of the screen.

Lesson 2: Variables and Strings

Variables

Variables make programs simpler to understand. The program uses the same expression changing the value of the variable to compute different things. This is much simpler than searching thru the code for each instance. Programmers create a name and use that name to refer to a variable, this is called an assignment statement: Name = Expression. An expression is assigned to a name using an assignment statement. The same expression can have different meanings at different times depending on when it is evaluated. A name can be any sequence of letters and numbers as long as it starts with a letter or underscore.

Variables are useful because they are easy to remember thus improving code readability, their values are easily changed and variables are a simple way to store values.

Is = really equal?

'=' does not mean the same thing as 'equal'. '=' is an assignment. A value (expression) is assigned to a name, it takes the value of the expression. In the equation 2 + 3 = 5 the sum of the left side is the same as the value on the right, this is an equation. My_name = 'Audrey' is an assignment. The expression on the right is assigned to the name on the left side of the '='.

Strings

Strings are a sequence of characters surrounded by quotes. This can be double or single quotes, but must be the same on both ends. Strings cannot be added to an integer but may be multiplied by one.


The number 2 is handled differently than the string "2". Code 2 + 2 produces a result of 4 while "2" + "2" produces "22".

Indexing Strings

We can extract subsequences from strings. This cannot be done with variables. <string> [<expression>], the expression is a number. The characters in a string are indexed from zero. The first letter is zero, 2nd is 1, etc. A subsequence of characters in a string begin from position start and end with position stop-1. <string> [<expression>:<expression>]


Finding Strings in Strings

Provides a way to find a sub string within a large string; find is a method. Finding strings in strings <string>.find(<string>) will provide the first occurrence in the search string where the target string appears. The parameter is the string and/or number inside the parenthesis of a find statement. A -1 result indicates the target string is not found and case does matter.

Lesson 3: Input ->Function -> Output

Using Procedures

A function is a procedure that performs a task or routine and returns a value. There are two types of functions, built in and those you can create yourself. Find, which was used in the prior lesson, is a built in function.


All functions begin with def (define). The def header is followed by a function name and indicates the number of input values to be expected.


The function is used, or executes a return statement when a procedure is called. The interpreter jumps int to the body of the procedure to run the code and will assign the string to the parameters, the values are passed in as inputs. 'a' is now the value passed in, which is 'audacity'. Return jumps back to where the procedure was called and provides the output or value for name. Without the return statement nothing is returned, the result is None.


Functions are valuable because they reduce repetition in code because they are reusable.

Making Decisions

When making decisions the following format is used:

Lesson 5: Structured Data: Lists and For Loops

Structured Data

Structured data is organized in a fixed field. A string or list is structured data. Strings are a sequence of variables like a name or word. A list is a sequence of anything, letters, words, numbers or characters. Lists are more powerful than strings.

Nested Lists are lists within lists, for example mixed_up['apple',3,'oranges',2,['alpha','beta']]. ['alpha','beta'] is nested within the mixed_up list.

Mutation and Aliases

Mutable
Lists are mutable meaning they may be changed after being created. The string is not modified it actually changes the value of the string by creating a new string. One thing to be cautious of is other variables that refer to that object.

Alias
Aliases have 2 different ways to refer to the same object. The example provided in class was James Bond is also known as '007'. Whatever happens to that person referred to as James Bond also happens to '007', the agent is the spy.

Note: you cannot mutate the value of a number.

Append and '+'

Append
Append is a method, like a procedure, used similarly to the way we use find. <list>.append(<element>). the object is appended in or added to the list.

'+'
'+' is similar to concatenation, adding the lists together to create one list.

len
len - length, works for any object that is a collection of things. len counts the number of elements in a list.

For Loop

Loops are a collection of things. The for loop is a simpler way to cycle thru all the elements in a list than to use a while loop with len. It is used when you have a section of code you want to repeat n number of times. For loop format is:


<name> in <list>:
    <block>

Index and In

Index
Index is a built in procedure. <list>.index(<value>) Index returns the first positon where <value> is found in <list> but will not provide a -1 if the value is not found, it will result in an error i.e.'3 is not in list' if looking for the value of 3.


In
In is another way to find a value within a list. <value>in<list>. If the <value> is in the <list>, output is True, otherwise the output is False. Another way to use in is <value>not in<list> which is the same as not <value>in<list> but is more in line with how it would be said in English.

Stage 3: Abstraction

Object Oriented Programming

What is Object Oriented Programming?

Object Oriented Programming (OOP) is a programming language that is based on objects which can be applied in a modular fashion. Objects are data structures that contain data. A programmer is able to manipulate objects and identify how they relate to each other. They are arranged in a hierarchy where a single object can be used in many different but related ways. One example is the fish vs salmon lesson. Salmon is an object (or instance) of the class fish. There are many types of fish, salmon is just one type. A specific fish, Mary, is an object as well; Mary is an object of the class salmon.

Instances are also known as objects. An object is one instance of a class. Each distinct logic sequence, or programmed procedure, is known as a method. There can be more than one method per object or class. And each method only has access to the data within it's object.

OOP uses modules which contain definitions and statements. There are built-in modules that can be imported and used in Python. The Python Standard Library is a great source for modules and their definitions as well as other aspects related to Python. Another source of information for Python programmers is the Google Python Style Guide which lists language and style rules.

OOP is a Valuable Tool

OOP is diverse, it provides countless way to manipulate objects. Inheritance allows for reduced repetition in programming. Objects make for simpler programming with the ability to break the program into smaller, more manageable pieces. Security is enhanced with OOP by placing data where it is not easily accessible by the rest of the system. Also, changes made to one object will not impact another.

Connections Made to Abstract Thinking

Abstract thinking is relational, being able to think about attributes and relationships separate from the objects that have those attributes or share those relationships. Object Oriented Programming uses relationships and attributes of items. HTML and CSS do the same. This type of thinking opens up the opportunity to group data in a way that reduces code by reusing it for similar classes, concepts, etc.

An example could be how you divide or group people. They could be divided or grouped by age, hair color, home town, or shoe size. There are endless opportunities for relating data. Object Oriented Programming provides a hierarchical method to organize and relate data into manageable chunks of data in a way that allows for safe and easy access.

Stage 4: Let Anyone Modify

Lesson 1: Servers

Requests

A client (web browser) sends a message to the server (web server) following protocal using HTTP (Hypertext Transfer Protocol). The message sent is a GET message in the form of a URL (Uniform Resource Locator). The URL is made up of the protocol, host and path. HTTP is the protocol, www.cnn.com is a host and / is the path (another example would be /money). The server accepts the message and runs some code. The server then responds with the contents of the requested object.

The request from your browser begins with a request line. The request line includes the method (type of request - most common are GET and POST), path - the actual document being requested, and version of HTTP. Since you are already connected to the host, which indicates where we are. The path is used for making the request.

Responses

The browser makes a request to the server then the server sends back a response. The response includes a status line which is made up of the status code and reason phrase. Headers are included in both requests and responses. Examples are Date and Content - Type and Length


Servers

The purpose of a server is to respond to HTTP requests. Documents are requested from the client via the web browser. Static and Dynamic responses are sent from the server. Static requests are pre-written forms or images. Dynamic responses are made on the fly. Most pages on the web are built dynamicly by web applications.

Lesson 2: Validation

The Importance of Validating Input

Security is paramount and using the escape and autoescape functions ensures that users are not able to hack into the sytem using java or html when entering data. Validation is a confirmation on the server side that we received what we expected to see. Input validation will improve a site's security. Escape is used to prevent unauthorized access by entering random text or code to alter a program. Using the escape functionality can also avoid promblems with a page that could make for a poor user experience. If the user was unable to link to the page, share or recieves an empty form. An error may require the user to reload the page in order to resubmit a form. These can be managed thru validation and proper coding. A redirect would make for a nicer user experience.

Templates and Abstraction

HTML templates are forms that can be used repeatedly, ultimately reducing duplicate code and saving time during updates since there is only one place to make changes instead of multiple throughout the code. This is the art of abstraction, creating models that are re-used so no rewriting is necessary. The html templates in this course were designed to request data from the user which was then rendered in Python.

Templates inheritance allows you to have one template that includes your header and footer and sub-templates that contain the content. The example in class showed how shopping_list.html and fizzbuzz.html inherit from base.html using a block.
